Introduction - PyTorch

PyTorch is a popular Deep Learning framework based on the Torch library. Software in Tesla Autopilot, Uber's Pyro, Hugging Face's Transformers and more are implemented using this framework. It is growing in popularity as most papers that are published in the field of machine learning especially in computer vision and natural language processing have been tested and implemented using it. As to why it's so popular, two main arguments hold:

  1. Tensor computing with strong acceleration via GPUs
  2. Deep neural networks built on an automatic differentation system using computational graphs
These make PyTorch incredibly fast, versatile, and efficient. Hopefuly throughout this post we'll get a chance to experience those perks ourselves.

PyTorch tensors

A tensor can be thought of as an n-dimensional array. It isn't 100% mathematically correct, since we would be talking about ranks but for understanding purposes I stick with this. Vectors are 1D tensors while matrices are 2D tensors. This term dominates all implementations of machine learning applications. Usually, tensors store parameters which interact with other tensors using special operations defined in the field of linear algebra, which you can understand easily without prerequisite knowledge. The main advantage of PyTorch tensors is that they can be operated on a CUDA-capable NVIDIA GPU which speeds up training and predictions immensely.

PyTorch modules

The PyTorch ecosystem contains 3 high-level modules and many more submodules. Here they are:

  1. Autograd module

    2. The method that belongs to this module called automatic differentiation is key for the tuning of parameters in neural networks. This method computes the gradients of the output with respect to each parameter in the neural network. It does so with the use of computational graphs where it, roughly speaking, "remembers" the operations you've done throughout your network and then replays it back to compute the gradients.

  2. Optim module

    3. torch.optim is a module that implements optimization algorithms used for building NNs. This speeds up building NNs significantly.

  3. nn module

    4. The nn module provides layers and tools to easily create the neural network architecture by just defining the layers of the network.

PyTorch also has many useful submodules such as data loading utilities which we will use below. Here's the Wikipedia example of constructing a simple Neural Network architecture using PyTorch:

You also saw the function forward which implements a simple forward pass. The backwards pass is the more computationally intense part. Going forward you only do matrix multiplications between weights and node values and you apply activation functions. The backward pass consists of updating gradients of the loss function with respect to each parameter of the Neural Network.

Configuring an environment - Anaconda

One of the biggest bottlenecks for beginners in ML is configuring environments and installing all requirements. Often you work on the "global" environment and install multiple versions of the same things which innevitably leads you to using the wrong version when developing. This not only curses your global scope, it also leads you down StackOverflow rabbit holes from which few escape.

The hands-down best way to get around this issue is to install a package manager. If you've installed any Python distribution successfully, you should also have the Package Installer for Python (abrv. pip). pip allows you to install Python packages/libraries on the go through a terminal. Before you start spamming "pip install x" in the terminal, a common warning is to make a virtual environment in which you will use these commands. A virutal environment allows you to solve the above-mentioned problem. It allows you to install packages in specific environments which don't "touch" your global scope, to which everything by default has access. For example, if you install Python into a virtual environment and don't install it in a global scope, running Python through cmd won't work if you aren't in that specific environment.

To see how to use virtual envs with pip, go somewhere else. I prefer conda, one of the most popular options for package management when working with multiple languages or need different versions of Python.

Setting up an IDE

Configuring an IDE with a working package environment such as the one we just built can be tricky. There are a lot of options for IDEs but some common ones are: Google Colab, PyCharm, Jupyter Notebook, VSCode.

I listed them in increasing configurational pain. Colab is the easiest one for environments but it is also in the cloud. You don't get that satisfying feeling of running something locally and seeing your machine sweat. Jupyter Notebook is run in the browser, which again isn't that satisfying but it does run from your own components. Colab might be the best for beginners for some time (until you have to get into environment mingling). Colab offers free GPUs & sometimes TPUs for use. There are a bunch of pros and cons for Colab which I encourage you search by yourself. I won't explain how to connect our new environment to each of these IDEs but that's also easily searchable online.
